Floats:

Basic flow of display:block elements are top to bottom.

display: float; will allow us to take the element out of this flow, and position left or right.

Floats must have width

Next block element's background will fill the space and its contents will wrap around the floated element

float: none; will turn element go back into the flow

content content content content content content content content content content content. Notice how the content wraps around the floated item even though element itself takes replaces the floated space entirely

Notice the float element does not break out of the parent container and aligns to the left most side of it. We can nest floated elements like this.

If you want to stack more items horizontally, have them float left too

However, since both 'sidebar' and 'content' are taken out of the flow, the next normal elements (footer) take place after the 'menu'. Remember, floated elements DO NOT add to the height of normal flow.

Clearing Floats:

Adding clear: float will force the element to treat previous floats as blocks

Content: Now I'm floating too
Clearing left will clear left floating elements only. Right will only clear right floats. Usually we want both. Also, clearing will also add the height of the floated elements back into the flow

Also, additional floated elements will move down if there aren't enough width spaces

We can fix this by adding 'overflow': 'hidden' on the float container.

overflow : hidden -- controls the content. With floats, it tells the container to consider the float heights

overflow can do more. It basically controls the content in the fixed height container. Consider texts spilling over the container. Overflow 'visible' will show the text beyond the container, 'hidden' will hide the text, 'scroll' will create a scroll, and 'auto' will created a scroll if needed '
